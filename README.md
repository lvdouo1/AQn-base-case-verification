# Verification code for base cases in augmented cubes

This repository contains the computer-assisted verification code for the base cases \(n=3,4\) of the edge-specified path-cycle cover theorem in augmented cubes under one faulty edge.

## Verified statement

Let \(AQ_n\) be the \(n\)-dimensional augmented cube. For \(n=3,4\), the program verifies the following statement:

For every fault set \(F\subseteq E(AQ_n)\) with \(|F|\le 1\), every nonfaulty edge \(e=uv\in E(AQ_n)\setminus F\), and every integer \(l\) satisfying
\[
2\le l\le 2^n-4,
\]
there exist in \(AQ_n-F\):

- a simple \(u\)-\(v\) path \(P\) of length \(l\);
- a cycle \(C\) of length \(2^n-l-1\);

such that
\[
V(P)\cap V(C)=\varnothing,
\qquad
V(P)\cup V(C)=V(AQ_n).
\]

Here both path length and cycle length are counted by edges.

## Files

- `verify_base_cases.py`  
  Main verification program.

## Requirements

The program uses only the Python standard library.

Recommended version:

```bash
Python 3.10+
```

## Usage

Verify both base cases \(n=3\) and \(n=4\):

```bash
python verify_base_cases.py
```

Verify only \(AQ_3\):

```bash
python verify_base_cases.py --n 3
```

Verify only \(AQ_4\):

```bash
python verify_base_cases.py --n 4
```

Export explicit witnesses to a JSON file:

```bash
python verify_base_cases.py --n 3 --dump witnesses_n3.json
python verify_base_cases.py --n 4 --dump witnesses_n4.json
```

## Output

For each \(n\), the program reports:

- the number of vertices of \(AQ_n\);
- the number of edges of \(AQ_n\);
- the number of fault sets checked;
- the admissible length range;
- the progress of the verification;
- whether the verification succeeds.

If a counterexample is found, the program prints:

- the dimension \(n\);
- the fault set \(F\);
- the specified edge \(e\);
- the length \(l\);

for which no required path-cycle cover is found.

If the `--dump` option is used, the program writes explicit witnesses to a JSON file. Each witness contains:

- the fault set;
- the specified edge;
- the path length;
- one valid path \(P\);
- one valid cycle \(C\).

## Implementation notes

The augmented cube \(AQ_n\) is generated by its standard Cayley graph representation on binary strings of length \(n\).

The verification proceeds by exhaustive search:

1. Generate \(AQ_n\).
2. Enumerate all fault sets \(F\) with \(|F|\le 1\).
3. For each fault set, enumerate every nonfaulty edge \(e=uv\).
4. For each admissible length \(l\), search for a \(u\)-\(v\) path \(P\) of length \(l\).
5. Check whether the remaining vertices induce a Hamiltonian cycle \(C\).

The search is feasible for the base cases \(n=3,4\).
